# 📊 Финансовый AI-ассистент

Консольный AI-ассистент для анализа финансовых данных компании. Отвечает на вопросы строго по данным из CSV-файла, объясняет расчёты и выявляет тренды.

---

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone <url>
cd <repo>

# 2. Создать виртуальное окружение
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать .env (опционально, но рекомендуется)
Создай файл `.env` в корне проекта (рядом с `finance_ai_assistant.py`).
Добавь в `.env`:
DEEPSEEK_API_KEY=твой_ключ
#Опционально (если нужно менять):
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat

# 5. Запустить
python finance_ai_assistant.py
```

## Архитектура

```
finance_ai_assistant.py      ← точка входа
finance_assistant/
├── data.py       ← загрузка CSV (авто-определение кодировки)
├── metrics.py    ← расчёт Revenue Growth, Operating/Net/Gross Margin
├── llm.py        ← LLM-интеграция (Anthropic → DeepSeek → fallback)
├── display.py    ← rich-интерфейс: таблицы, панели, чат
└── cli.py        ← основной цикл чата с историей диалога
financial_data.csv            ← данные компании 2005–2024
```

### Потоки данных

1. `data.py` читает `financial_data.csv` → список словарей  
2. `metrics.py` рассчитывает метрики за каждый год + сводную статистику  
3. `metrics.py → format_context()` сериализует всё в JSON для LLM  
4. `llm.py` передаёт JSON + вопрос + историю диалога в LLM  
5. `display.py` красиво рендерит ответ в терминале через `rich`

### Режимы LLM (приоритет по убыванию)

| Приоритет | Условие | Режим |
|-----------|---------|-------|
| 1 | `DEEPSEEK_API_KEY` задан | DeepSeek через OpenAI-совместимый API |
| 2 | Ключ не задан | Rule-based fallback (без сети) |

### Вычисляемые метрики

| Метрика | Формула |
|---------|---------|
| Revenue Growth | `(revenue[t] - revenue[t-1]) / revenue[t-1]` |
| Gross Margin | `(revenue - cogs) / revenue` |
| Operating Margin | `(revenue - cogs - opex) / revenue` |
| Net Margin | `net_income / revenue` |

## Возможности интерфейса

- **Красивая таблица** всех метрик за 20 лет с цветовым кодированием  
- **Сводная панель** с ключевыми показателями  
- **Многоходовой чат** — ассистент помнит контекст диалога  
- **Команды**: `help` (подсказки), `clear` (сбросить историю), `exit` (выход)  
- **Spinner** — индикатор ожидания при обращении к LLM  

---

## Промпт-стратегия

Системный промпт явно запрещает модели придумывать числа и требует:
- Использовать только значения из переданного JSON-контекста  
- Показывать формулы расчётов с подстановкой реальных чисел  
- Структурированный вывод: факт → анализ → вывод  
- Отмечать аномалии, тренды, переломные моменты

## Примеры запросов и ответов

**Вопрос:** `В каком году был самый быстрый рост выручки?`

> **Ответ:**
> Самый быстрый рост выручки зафиксирован в **2006 году** — ≈ 29.2%.
>
> ▶ Расчёт: (155 000 − 120 000) / 120 000 × 100 = 29.17%
>
> → Это единственный год с ростом выше 29%. В последующие годы темп роста стабилизировался на уровне 17–18%.

---

**Вопрос:** `Как изменялась прибыльность компании со временем?`

> **Ответ:**
> Прибыльность компании исключительно стабильна на протяжении 20 лет.
>
> ▶ Операционная маржа: 24.49% – 25.09% (среднее 24.80%)
> ▶ Чистая маржа: 24.49% – 25.09% (среднее 24.80%)
>
> → Компания сохраняет ~25% чистой маржи независимо от роста выручки — это признак высокой операционной эффективности и устойчивой бизнес-модели.

---

**Вопрос:** `Сравни 2010 и 2020 год по всем показателям`

> **Ответ:**
> ▶ Выручка: $340 000 (2010) → $1 960 000 (2020), рост в 5.76×
> ▶ Чистая прибыль: $84 000 → $491 000, рост в 5.85×
> ▶ Операционная маржа: 24.71% vs 24.85% — почти идентичны
> ▶ Рост выручки в 2020: 17.4%
>
> → За десятилетие компания масштабировалась почти в 6 раз, не потеряв рентабельности — редкое достижение.

---

**Вопрос:** `Когда компания росла медленнее всего?`

> **Ответ:**
> Самый медленный рост выручки — **2008 год** (22.5%) и **2009 год** (16.3%).
>
> ▶ 2008: $200 000 → $245 000 (+22.5%)
> ▶ 2009: $245 000 → $285 000 (+16.3%)
>
> → Замедление в 2009 году, вероятно, отражает последствия глобального финансового кризиса 2008–2009 годов. Это единственный период с заметным снижением темпов роста.
